# Migration `v1.0.3-vault-language`

This slash-command is invoked automatically by `/update-vault` when the vault is in version < 1.0.3. It updates the **user-owned** files (`CLAUDE.md` and `.claude/agents/<domain>-expert.md`) with the `vault_language` directive introduced in v1.0.3.

Without this script, `/update-vault` propagates the upstream-tracked files (`.claude/rules/*`, `.claude/commands/*`, `wiki/decisions/<3 ADRs>`) in EN, but leaves `CLAUDE.md` and the per-domain agents with the FR-hardcoded language line ‚Äî the source-language-vs-vault-language regression stays active for users who don't manually edit those files.

## Context

Before v1.0.3, `CLAUDE.md.tpl` had `Fran√ßais.` hardcoded in the *Writing principles* section, and `domain-expert.md.tpl` had `"titres en fran√ßais"` in the *Wiki rules* section. Consequence: an EN user bootstrapped a vault, and every page produced by `/ingest` came out in French regardless of the source language.

v1.0.3 introduced:

- A new `{{vault_language}}` placeholder (29th global) capturing the language detected at the bootstrap interview.
- A directive in `CLAUDE.md.tpl` and `domain-expert.md.tpl`: *"Source language is independent of vault language. Sources can be in any language; wiki pages must always be written in the vault language."*

For new vaults, this is handled automatically at bootstrap. For **existing** vaults migrating from v1.0.x to v1.0.3, this script does the equivalent retrofit.

`CLAUDE.md` and `.claude/agents/<domain>-expert.md` are **user-owned** (customized at bootstrap, potentially enriched afterwards). The migration **never rewrites them silently** ‚Äî every change goes through `AskUserQuestion`.

## Workflow

### 1. Ask the user for the vault language

Always ask, never guess silently. The user may want to change the vault language at this occasion (e.g. someone who was on FR by default but wants to switch to EN).

First, **detect the current language** by reading `CLAUDE.md`:

- Look for the *Writing principles* section (`## Principes d'√©criture`, `## Writing principles`, or any localized variant).
- Inside it, look for a line that declares the language. Typical patterns:
  - `Fran√ßais.` or `Fran√ßais ‚Äî ...`
  - `English.` or `English ‚Äî ...`
  - `Vault language: <X>` (already migrated)
  - The full new directive `Vault language: **<X>** ‚Äî every wiki page is written in this language...`
- If a language is detected, store it as `current_language` and use it as the recommended pre-selected option.
- If nothing is detected, fall back to no recommendation.

Then ask via `AskUserQuestion`:

```json
{
  "questions": [{
    "question": "Which language do you want to use for this vault? (every wiki page produced by /ingest will be written in this language, regardless of source language)",
    "header": "Vault language",
    "multiSelect": false,
    "options": [
      {"label": "English", "description": "All wiki pages in English."},
      {"label": "Fran√ßais", "description": "Toutes les pages wiki en fran√ßais."},
      {"label": "Espa√±ol", "description": "Todas las p√°ginas wiki en espa√±ol."},

      {"label": "Other", "description": "Free-text another language label (e.g. Êó•Êú¨Ë™û, Portugu√™s, Italiano)."}

```

If `current_language` was detected, mark its option label with `(Recommended ‚Äî currently detected)` and place it first.

If the user picks `Other`, prompt for a free-text language label and store it as `vault_language`.

If the user picks a language **different** from `current_language`, note it explicitly. The migration will be flagged as a *language change* (vs *language label retrofit*) in the commit message and `wiki/log.md`.

Store the resolved value in `vault_language`.

### 2. Migrate `CLAUDE.md`

Locate the *Writing principles* section in `CLAUDE.md`. Look for the language line. Examples that should be replaced:

- `- Fran√ßais. Termes techniques en VO si l'usage VO est dominant.`
- `- English. Technical terms in VO when the VO usage is dominant.`
- Any single-language line in that section.

Propose the replacement (rendered in the same language as the surrounding `CLAUDE.md` to keep the file's style coherent ‚Äî auto-detect from the section heading or the rest of the prose):

If the surrounding language is **English**:

```markdown
- Vault language: **<vault_language>** ‚Äî every wiki page is written in this language, regardless of the source's original language. Technical terms in VO when the VO usage is dominant.
```

If the surrounding language is **Fran√ßais**:

```markdown
- Langue du vault : **<vault_language>** ‚Äî toute page wiki est r√©dig√©e dans cette langue, quelle que soit la langue de la source d'origine. Termes techniques en VO si l'usage VO est dominant.
```

For other languages, translate the directive accordingly while keeping the placeholder substitution `<vault_language>` resolved.

If no language line is found in *Writing principles*, propose to **insert** the new directive at the top of the section.

Show a clean diff (3 lines of context before / after) and ask via `AskUserQuestion`:

```json
{
  "questions": [{
    "question": "Apply this CLAUDE.md change?",
    "header": "CLAUDE.md",
    "multiSelect": false,
    "options": [
      {"label": "‚úÖ Apply", "description": "Writes the change. The rest of CLAUDE.md is untouched."},
      {"label": "‚úèÔ∏è Edit manually", "description": "Shows the diff and lets you handle it. The migration does not touch the file."},

    ]
  }]
}
```

Apply only if the user picks **Apply**.

### 3. Migrate each `<domain>-expert.md`

List `.claude/agents/*.md`. For each file, check if it's a domain-expert agent (filter heuristic):

- Frontmatter contains `name: <slug>-expert` AND `description: ...expert...` ‚Üí it's a domain-expert agent.
- Otherwise (e.g. `compress-bb.md`, `extract-range-grid.md`, or any custom slash-command-style agent) ‚Üí **skip**.

For each domain-expert agent, locate the line that declares writing language. Typical patterns:

- `- Les pages sont en \`kebab-case.md\`, les titres en fran√ßais (termes VO si usage consacr√©).`
- `- Pages are in \`kebab-case.md\`, titles in English (VO terms if standard usage).`
- Any single-language declaration in the *Wiki rules / R√®gles du wiki* section.

Propose the replacement (rendered in the agent file's language ‚Äî typically same as `CLAUDE.md`'s language, but agents may have been customized differently). For an EN agent file:

```markdown
- Pages are in `kebab-case.md`. **Writing language**: titles and bodies in the vault language (`<vault_language>`, declared in `CLAUDE.md`), VO terms when usage is established. **The original source language has no incidence**: an EN source ingested into a FR vault produces FR pages; a FR source ingested into an EN vault produces EN pages. Quote source passages verbatim in their original language between quotes, translate your commentary / synthesis into the vault language.
```

For an FR agent file, render the directive in French (same shape, translated content).

Before iterating, ask via `AskUserQuestion` how the user wants to handle the batch:

```json
{
  "questions": [{
    "question": "How do you want to migrate the per-domain agents (N files detected)?",
    "header": "Agent migration",
    "multiSelect": false,
    "options": [
      {"label": "‚úÖ Apply to all", "description": "Apply the same change to every <domain>-expert.md without per-file confirmation. Recommended if you haven't customized the writing-language line of each agent."},
      {"label": "üîç Review one by one", "description": "Show a diff and ask for each file. Recommended if some agents have non-standard writing-language wording."},
      {"label": "‚è≠Ô∏è Skip all", "description": "Don't touch any agent. Migration will be re-proposed at next /update-vault."}
    ]
  }]
}
```

If **Apply to all**: apply the replacement to every detected agent file in one go, log the count.

If **Review one by one**: for each file, show the diff and ask Apply / Edit manually / Skip.

If **Skip all**: don't touch any agent; record this in the log.

### 4. Logging

Append to `wiki/log.md`:

```markdown
## [YYYY-MM-DD] migrate | v1.0.3-vault-language

- Vault language resolved: **<vault_language>** {{ "(changed from " + current_language + ")" if current_language and current_language != vault_language else "" }}.
- CLAUDE.md: applied / edited manually / skipped.
- Agents migrated: N applied / M edited manually / K skipped (out of <total> domain-expert agents detected).
- Custom agents skipped (non-domain-expert): <list of names if any>.
```

### 5. Commit

If at least one file was modified by this migration:

```bash
git add CLAUDE.md .claude/agents/*.md wiki/log.md
git commit -m "chore: migrate CLAUDE.md and agents to v1.0.3 vault_language directive"
```

If the user changed the vault language (vs the previously-detected one), add a second line to the commit message:

```
Vault language: <previous> ‚Üí <new>.
```

If nothing was modified (everything skipped), do not commit.

### 6. Verdict for `/update-vault`

If **all** proposed changes were Applied (CLAUDE.md + every detected agent) ‚Üí migration is **complete** ‚Üí `/update-vault` is allowed to bump `.claude/template-version` to 1.0.3.

If **any** change was skipped or edited manually ‚Üí migration is **partial** ‚Üí `/update-vault` must NOT bump `.claude/template-version`. Migration will be re-proposed at the next `/update-vault` (and will skip what was already applied ‚Äî see *Idempotence* below).

## Idempotence

The migration is idempotent at the file level:

- For `CLAUDE.md`: if the *Writing principles* section already contains the new `Vault language: ...` directive, skip the proposal (no duplicate insertion). Detect via regex on the directive shape.
- For agents: if the agent's writing-language line already mentions *"original source language has no incidence"* (or equivalent translation), skip.

This means a partial migration can be safely re-run by the next `/update-vault` without double-applying.

## Preserving user customizations

Both `CLAUDE.md` and `.claude/agents/<domain>-expert.md` may contain user customizations (sections added beyond the template, refinements, custom rules). The migration touches **only the writing-language line** in the *Writing principles* / *Wiki rules* section. Any other customization is preserved intact ‚Äî this script never deletes or rearranges sections.

If, in some edge case, the user has moved or renamed the *Writing principles* section, the migration will fail to locate it and report:

> ‚öÝÔ∏è Could not locate the Writing principles section in `<file>`. Please apply the change manually using the snippet shown, then re-run `/update-vault`.

Then offer Edit manually as the only remaining option for that file.

## Notes

- This is the second example of the `scripts/migrations/v<X>-*.md` pattern (after `v1.0.2-claude-md-slim.md`). The pattern is stable: one slash-command per breaking version, invoked by `/update-vault`, always interactive.
- Unlike `v1.0.2-claude-md-slim.md`, this migration does not restructure the file ‚Äî it only updates a single directive line per touched file. The risk surface is small.
- For users who genuinely want a single-language vault and never want the multilingual directive: pick **Skip** at every prompt. The vault keeps its previous wording, and `/update-vault` won't bump `.claude/template-version` until they decide otherwise.
