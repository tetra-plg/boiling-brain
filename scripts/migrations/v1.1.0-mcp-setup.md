# Migration `v1.1.0-mcp-setup`

This slash-command is invoked automatically by `/update-vault` when the vault is in version < 1.1.0. It installs the MCP + hooks stack introduced in v1.1:

- MCP server `boiling-brain-wiki` (FastMCP, 5 tools: `scan_domain`, `preview_page`, `read_page`, `search_wiki`, `drop_to_raw`).
- Global `Stop` hook (`~/.claude/settings.json`) that detects commits + modified files at end of session.
- Instructions block in `~/.claude/CLAUDE.md` (global) so Claude Code uses the MCP from any project.
- "Session start" section in the vault's `CLAUDE.md`, so that on the next launch Claude proposes `/compress-bb` or `/ingest` based on the signals left in `cache/`.

## Context

v1.1 introduces a channel to access the vault from any project (via user-scope MCP) and a session lifecycle loop to stop losing the learnings of a working session. Both mechanics require mutations on **user-owned global files** (`~/.claude/settings.json`, `~/.claude/CLAUDE.md`) — so an **interactive and idempotent** migration is mandatory.

The vault's `CLAUDE.md` (generated at bootstrap from `CLAUDE.md.tpl`) does not have the "Session start" section either in vaults bootstrapped before v1.1. Without that section, Claude ignores the signals `cache/.session-pending` and `cache/.pending-ingest` even if the Stop hook writes them correctly.

## Workflow

### 1. Announce what's new

Before any action, present to the user what the migration will do:

> **v1.1.0 — MCP + hooks + /compress-bb**
>
> This migration installs:
> - An MCP server `boiling-brain-wiki` accessible from **any project** (`scan_domain`, `preview_page`, `read_page`, `search_wiki`, `drop_to_raw`).
> - A global `Stop` hook that detects productive sessions and writes `cache/.session-pending` in the vault.
> - A "Session start" section in the vault's `CLAUDE.md` so that Claude proposes `/compress-bb` or `/ingest` on the next launch.
>
> Mutations performed:
> - MCP server registered via `claude mcp add -s user` (user-scope, not stored in `settings.json`).
> - `~/.claude/settings.json` (`hooks.Stop` only).
> - `~/.claude/CLAUDE.md` (`<!-- boiling-brain-wiki-mcp -->` block).
> - `<vault>/CLAUDE.md` (`## Session start` section).
> - `pipx install fastmcp` if absent (fallback `pip install --user`).
>
> Everything is idempotent and rollbackable.

### 2. AskUserQuestion — global decision

```json
{
  "questions": [{
    "question": "Enable the v1.1 MCP + hooks stack?",
    "header": "v1.1 setup",
    "multiSelect": false,
    "options": [
      {"label": "✅ Enable", "description": "Runs setup-mcp.sh (mutates settings.json + global CLAUDE.md) and patches the vault CLAUDE.md. Idempotent."},
      {"label": "✏️ Patch vault CLAUDE.md only", "description": "Skip the MCP server and the global Stop hook. Just add the Session start section to the vault's CLAUDE.md. Useful if you already installed MCP another way or don't want any global mutation."},
      {"label": "⏭️ Skip for this session", "description": "Touch nothing. The migration will be re-proposed at the next /update-vault. .claude/template-version stays at 1.0.x."}
    ]
  }]
}
```

### 3. Branch **"Enable"** — setup-mcp.sh + vault CLAUDE.md patch

#### 3.1. Run `setup-mcp.sh`

```bash
bash scripts/setup-mcp.sh --vault-path "$(pwd)"
```

The script (existing, shipped by v1.1) does:
- Checks Python 3 and the `claude` command (Claude Code CLI).
- Installs `fastmcp` if absent — pipx is prioritized (clean, isolated venv), pip --user as fallback. If both fail (PEP 668 + no pipx), explicit error.
- Resolves the Python interpreter that can actually `import fastmcp`: when pipx is used, it's the python from the pipx venv (`$PIPX_LOCAL_VENVS/fastmcp/bin/python`), **not** the system `python3` — this is the fix for the import-fastmcp error on PEP 668 systems.
- Registers the MCP server via `claude mcp add -s user boiling-brain-wiki -e WIKI_PATH=<vault> -- <pipx-venv-python> <vault>/scripts/mcp-wiki.py` (idempotent — detects via `claude mcp get`). User-scope means the server is visible from every project.
- Adds the `Stop` hook (command `bash <vault>/scripts/check-session-activity.sh`) to `~/.claude/settings.json` (idempotent — detects the path already present).
- Appends a block with marker `<!-- boiling-brain-wiki-mcp -->` to `~/.claude/CLAUDE.md` (idempotent — detects the marker).
- `chmod +x` on `scripts/check-session-activity.sh`.

Capture the script's stdout and present it to the user. If the fastmcp install or `claude mcp add` fails, stop and report.

#### 3.2. Patch the vault's `CLAUDE.md`

Read the vault's `CLAUDE.md`. If the `## Session start` section (or a close title: `## Session start (cache/ signals)`, or any localized variant such as `## Démarrage de session`) already exists → do nothing (idempotence, message "Section already present, skip").

Otherwise, insert the following block **before** the `## Writing principles` section (or, failing that, at the end of the file). Render the block in the vault language declared in `CLAUDE.md` — the snippet below is the EN reference; translate to the vault language if different.

```markdown
## Session start (`cache/` signals)

At the start of each session, check the signals left in `cache/`:

- **`cache/.pending-ingest`**: one or more paths waiting to be ingested (dropped via the MCP `drop_to_raw` or by another vault). Propose `/ingest <path>` for each entry. Do not delete the file — wait for the user's confirmation.
- **`cache/.session-pending`**: the previous session had unjournaled changes (commits + modified files detected by the `Stop` hook). Propose `/compress-bb <slug>` to archive the journal into `raw/notes/sessions/`. Delete the file after proposal.

These checks are silent if the files are absent.
```

Before writing, present the diff via `AskUserQuestion` (3 options: apply / edit manually / skip), like the v1.0.2 migration.

#### 3.3. Commit

If setup-mcp.sh OK + section added:

```bash
git add CLAUDE.md
git commit -m "chore: migrate to v1.1.0 (MCP + SessionStart signals)"
```

Only `CLAUDE.md` is added (setup-mcp.sh does not touch any vault file).

#### 3.4. Confirm

Display:

> ✅ Migration v1.1.0 applied.
> - MCP `boiling-brain-wiki` registered (user scope, via `claude mcp`).
> - Global `Stop` hook registered in `~/.claude/settings.json`.
> - "Session start" section added to `<vault>/CLAUDE.md`.
>
> **Restart Claude Code** to load the MCP server and hooks.
>
> Test with: `/mcp` (should list `boiling-brain-wiki`).

### 4. Branch **"Patch vault CLAUDE.md only"**

Same as 3.2 + 3.3 + 3.4 but without 3.1. The commit message becomes:

```
chore: add v1.1 SessionStart signals section to CLAUDE.md
```

### 5. Branch **"Skip"**

Do nothing. State:

> Migration `v1.1.0-mcp-setup` skipped. `.claude/template-version` stays on the previous version. The migration will be re-proposed at the next `/update-vault`.

## Idempotence

- **`setup-mcp.sh`** is idempotent by construction: it detects entries already present (MCP server via `claude mcp get`, Stop hook command, global CLAUDE.md marker).
- **The vault `CLAUDE.md` patch** is idempotent via detection of the `## Session start` heading (case-insensitive + tolerance for the `(cache/ signals)` suffix and localized variants like `## Démarrage de session`).
- If the user re-runs `/update-vault` after a partial migration, the migration re-detects what is already done and only proposes the remaining actions.

## Preserving customizations

- The vault's `CLAUDE.md` is never rewritten as a whole — only a new block is inserted at an identified location.
- If the user already has a section with a close-but-not-exact title, signal the potential conflict via `AskUserQuestion` rather than writing silently.
- `~/.claude/settings.json` and `~/.claude/CLAUDE.md` (globals) are mutated via targeted merge (never wholesale rewrite) — see `setup-mcp.sh`.

## Rollback

- **MCP server**: the entry is no longer in `settings.json` (since it's registered via `claude mcp add`). Remove it with `claude mcp remove boiling-brain-wiki -s user`.
- **Stop hook in `~/.claude/settings.json`**: edit the file manually to remove the entry under `hooks.Stop` whose command points to `<vault>/scripts/check-session-activity.sh`.
- **Block in `~/.claude/CLAUDE.md`**: delete the block between the two `<!-- boiling-brain-wiki-mcp -->` markers.
- **Section in `<vault>/CLAUDE.md`**: `git revert` the dedicated commit, or edit manually.
- **`fastmcp`** (pipx or pip --user): `pipx uninstall fastmcp` (or `pip uninstall fastmcp`) if desired (harmless to leave).

The full rollback procedure in pre-release testing context is in [`docs/local-pre-release-testing.md`](../../docs/local-pre-release-testing.md) Phase 5.
